**src/utils/text.ts:**

Export the following utility functions:

`escapeHtml(text: string): string`
- Replace `&` with `&amp;`, `<` with `&lt;`, `>` with `&gt;`
- Per Pitfall 3 from research: always escape user content before embedding in HTML templates

`truncateText(text: string, maxLength: number, suffix?: string): string`
- Default suffix is `'\n... (truncated)'`
- If text.length <= maxLength, return as-is
- Otherwise return text.slice(0, maxLength - suffix.length) + suffix

`splitForTelegram(text: string): { inline: true; text: string } | { inline: false; text: string }`
- Per user decision on smart output splitting:
  - If text.length <= 400: return `{ inline: true, text }` (short output -- inline in message)
  - If text.length <= 4000: return `{ inline: true, text }` (medium -- code block, still inline)
  - If text.length > 4000: return `{ inline: false, text }` (long -- will be sent as .txt attachment)
- The 4000 threshold leaves room for message template wrapping before hitting Telegram's 4096 limit

Export the following formatting functions:

`formatSessionStart(session: SessionInfo, isResume: boolean): string`
- If isResume: "🔄 <b>Session resumed</b> in <code>{cwd}</code>"
- If new: "🟢 <b>Session started</b> in <code>{cwd}</code>\nModel: <code>{session model if available or 'unknown'}</code>"
- Escape cwd with escapeHtml

`formatSessionEnd(session: SessionInfo, reason: string): string`
- Per user decision: post a final summary before closing topic
- Format: "✅ <b>Session ended</b> ({reason})\n\n📊 <b>Summary:</b>\n• Duration: {calculated from startedAt to now}\n• Tool calls: {session.toolCallCount}\n• Files changed: {session.filesChanged count}"
- If filesChanged has items, list up to 10 with `<code>` tags, then "(+ N more)" if > 10
- Calculate duration as human-readable (e.g., "12 minutes", "1 hour 23 minutes")

`formatToolUse(payload: PostToolUsePayload): { text: string; attachment?: { content: string; filename: string } }`
- Return both a formatted text AND optionally an attachment for long outputs
- Per user decision on tiered verbosity:

  **Compact one-liners (reads/searches):**
  - tool_name "Read": `"👁 Read <code>{filePath}</code>"`
  - tool_name "Glob": `"🔍 Glob <code>{pattern}</code> ({count} matches)"`
  - tool_name "Grep": `"🔍 Grep <code>{pattern}</code> in <code>{path}</code>"`

  **Detailed blocks (edits/bash):**
  - tool_name "Write": `"📝 <b>Write</b> <code>{filePath}</code>\n<pre>{truncated content preview, first 500 chars}</pre>"`
  - tool_name "Edit": Per user decision on diffs:
    `"📝 <b>Edit</b> <code>{filePath}</code>\n<pre>- {old_string}\n+ {new_string}</pre>"`
    Use `escapeHtml` on all content. If old_string/new_string combined > 1000 chars, truncate each to 500.
  - tool_name "MultiEdit": Same as Edit but note "({count} edits)"
  - tool_name "Bash": `"💻 <b>Bash</b>\n<pre><code>{command}</code></pre>"`
    If tool_response has output, use `splitForTelegram`:
    - If inline and short: append as `<blockquote>{output}</blockquote>`
    - If inline and medium: append as `<pre>{output}</pre>`
    - If not inline: set attachment = `{ content: outputText, filename: 'bash-output.txt' }`

  **Default for unknown tools:**
  `"🔧 <b>{toolName}</b>"` with tool_input as JSON code block (truncated to 500 chars)

- Extract tool_input fields defensively: `payload.tool_input?.file_path as string | undefined`, etc.
- Extract tool_response fields: `payload.tool_response?.content as string | undefined`, etc.
- Always escapeHtml on all user-supplied content

**src/bot/rate-limiter.ts:**

Per research Pattern 3 (Layer 2): custom per-session debounce queue.

Export `class MessageBatcher`:

**Constructor:**
- Takes `sendFn: (threadId: number, html: string) => Promise<void>` -- the function to call when flushing
- Takes `sendDocFn: (threadId: number, content: string, filename: string, caption?: string) => Promise<void>` -- for file attachments
- Takes `batchWindowMs: number = 2000` (2-second debounce window per Claude's discretion)

**Private state:**
- `queues: Map<number, { messages: string[]; attachments: Array<{content: string; filename: string; caption?: string}>; timer: NodeJS.Timeout | null }>` -- keyed by threadId

**Public methods:**

`enqueue(threadId: number, message: string, attachment?: { content: string; filename: string; caption?: string }): void`
- Add message to the threadId's queue
- If attachment, add to attachments array
- If timer exists, clear it
- Set new timer for batchWindowMs ms, calling `this.flush(threadId)`

`enqueueImmediate(threadId: number, message: string): void`
- For messages that should NOT be batched (session start, session end, bot restart notices)
- Flush any pending messages for this threadId first, then send immediately via sendFn

`flush(threadId: number): Promise<void>`
- Get the queue for threadId
- If empty, return
- Combine all messages with `'\n\n'` separator
- If combined text > 4096, split into multiple sends of <= 4096 each (split on message boundaries, not mid-message)
- Call sendFn for each text chunk
- Call sendDocFn for each attachment
- Clear the queue and timer

`shutdown(): void`
- Flush all pending queues immediately (for graceful shutdown)
- Clear all timers
