<task type="auto">
  <name>Task 2: Define type contracts and config loader</name>
  <files>src/types/hooks.ts, src/types/config.ts, src/types/sessions.ts, src/config.ts</files>
  <action>
Create all type definitions and the config loader. These are the contracts every other plan depends on.

1. Create `src/types/hooks.ts` -- TypeScript types for Claude Code hook payloads:
   ```typescript
   // Base payload present in all hook events
   export interface HookPayload {
     session_id: string;
     transcript_path: string;
     cwd: string;
     permission_mode: string;
     hook_event_name: string;
   }

   export interface SessionStartPayload extends HookPayload {
     hook_event_name: 'SessionStart';
     source: 'startup' | 'resume' | 'clear' | 'compact';
     model: string;
   }

   export interface SessionEndPayload extends HookPayload {
     hook_event_name: 'SessionEnd';
     reason: string;
   }

   export interface PostToolUsePayload extends HookPayload {
     hook_event_name: 'PostToolUse';
     tool_name: string;
     tool_input: Record<string, unknown>;
     tool_response: Record<string, unknown>;
     tool_use_id: string;
   }
   ```

2. Create `src/types/config.ts` -- Configuration schema:
   ```typescript
   export interface AppConfig {
     telegramBotToken: string;
     telegramChatId: number;      // Telegram group chat ID (negative number for supergroups)
     hookServerPort: number;       // Default: 3456
     hookServerHost: string;       // Default: '127.0.0.1'
     dataDir: string;              // Default: './data'
   }
   ```

3. Create `src/types/sessions.ts` -- Session data types:
   ```typescript
   export interface SessionInfo {
     sessionId: string;
     threadId: number;             // Telegram message_thread_id
     cwd: string;                  // Project directory
     topicName: string;            // Display name in Telegram
     startedAt: string;            // ISO timestamp
     status: 'active' | 'closed';
     transcriptPath: string;       // Path to JSONL transcript
     toolCallCount: number;        // For final summary
     filesChanged: Set<string> | string[];  // For final summary (serialized as array in JSON)
   }
   ```
   Note: When persisting to JSON, `filesChanged` should be serialized as `string[]`. The `Set<string>` type is for in-memory use to deduplicate file paths.

4. Create `src/config.ts` -- Config loader:
   - Import and call `dotenv/config` at the top (side-effect import)
   - Export `loadConfig(): AppConfig` function
   - Read from `process.env`: `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`, `HOOK_SERVER_PORT`, `HOOK_SERVER_HOST`, `DATA_DIR`
   - Validate: `TELEGRAM_BOT_TOKEN` and `TELEGRAM_CHAT_ID` are REQUIRED. If missing, print a clear error message listing the missing vars and how to set them (reference .env.example), then call `process.exit(1)` -- per user decision "show clear error with setup steps, no interactive wizard"
   - Defaults: port=3456, host='127.0.0.1', dataDir='./data'
   - Parse `TELEGRAM_CHAT_ID` as integer with `parseInt()`
   - Return typed `AppConfig` object
  </action>
