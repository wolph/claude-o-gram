Create the hook auto-installer and the main entry point that wires everything together.

**src/utils/install-hooks.ts:**

Per user decision: "Bot auto-installs Claude Code hooks by writing hook config to Claude Code's settings on first run."
Per research Pattern 4.

Export `function installHooks(port: number): void`:

1. Import `readFileSync, writeFileSync, existsSync, mkdirSync` from `node:fs`
2. Import `join` from `node:path`
3. Import `homedir` from `node:os`

4. Compute paths:
   - `claudeDir = join(homedir(), '.claude')`
   - `settingsPath = join(claudeDir, 'settings.json')`

5. Ensure `~/.claude/` directory exists

6. Read existing settings.json if it exists, otherwise start with empty object

7. Define the hook configuration for SessionStart, SessionEnd, and PostToolUse:
   ```
   const baseUrl = `http://127.0.0.1:${port}`;
   const hookConfig = {
     SessionStart: [{ hooks: [{ type: 'http', url: `${baseUrl}/hooks/session-start`, timeout: 10 }] }],
     SessionEnd: [{ hooks: [{ type: 'http', url: `${baseUrl}/hooks/session-end`, timeout: 10 }] }],
     PostToolUse: [{ hooks: [{ type: 'http', url: `${baseUrl}/hooks/post-tool-use`, timeout: 10 }] }],
   };
   ```

8. Merge into settings: `settings.hooks = { ...settings.hooks, ...hookConfig }`
   - This preserves any existing hooks for other events while overwriting our three

9. Write back with `JSON.stringify(settings, null, 2)`

10. Log a message indicating hooks were installed/updated and the settings path

**src/index.ts:**

This is the main entry point. Per research Pattern 1: single-process dual server.

Export `async function main(): Promise<void>` and call it at module level.

**Startup sequence:**

1. Import all modules:
   - `loadConfig` from `./config.js`
   - `SessionStore` from `./sessions/session-store.js`
   - `createHookHandlers`, `HookCallbacks` from `./hooks/handlers.js`
   - `createHookServer` from `./hooks/server.js`
   - `createBot` from `./bot/bot.js`
   - `TopicManager` from `./bot/topics.js`
   - `MessageBatcher` from `./bot/rate-limiter.js`
   - `formatSessionStart`, `formatSessionEnd`, `formatToolUse` from `./bot/formatter.js`
   - `installHooks` from `./utils/install-hooks.js`
   - `join` from `node:path`

2. Load config: `const config = loadConfig()`

3. Auto-install hooks: `installHooks(config.hookServerPort)`
   - Per user decision: auto-install on every startup (idempotent)

4. Initialize session store: `const sessionStore = new SessionStore(join(config.dataDir, 'sessions.json'))`

5. Create bot: `const bot = await createBot(config)`

6. Create topic manager: `const topicManager = new TopicManager(bot, config.telegramChatId)`

7. Create message batcher with send functions wired to topic manager:
   ```
   const batcher = new MessageBatcher(
     (threadId, html) => topicManager.sendMessage(threadId, html),
     (threadId, content, filename, caption) => topicManager.sendDocument(threadId, content, filename, caption),
   );
   ```

8. **Define HookCallbacks -- THIS IS THE CRITICAL WIRING:**

   `onSessionStart: async (session) => { ... }`
   - Determine if this is a resume: check if session was created from a resume source (pass through a flag or check session store)
   - If resume and session already has a threadId > 0:
     - Reopen topic: `await topicManager.reopenTopic(session.threadId, session.topicName)`
     - Send resume message: `batcher.enqueueImmediate(session.threadId, formatSessionStart(session, true))`
   - If new session:
     - Create topic: `const threadId = await topicManager.createTopic(session.topicName)`
     - Update session store with threadId: `sessionStore.updateThreadId(session.sessionId, threadId)`
     - Update the session object's threadId for the rest of this callback
     - Send start message: `batcher.enqueueImmediate(threadId, formatSessionStart(session, false))`

   `onSessionEnd: async (session) => { ... }`
   - Re-fetch session from store to get latest toolCallCount/filesChanged
   - Format final summary: `const summary = formatSessionEnd(session, 'completed')`
   - Send immediately (not batched): `batcher.enqueueImmediate(session.threadId, summary)`
   - Close topic: `await topicManager.closeTopic(session.threadId, session.topicName)`

   `onToolUse: async (session, payload) => { ... }`
   - Format the tool use: `const result = formatToolUse(payload)`
   - Enqueue the message (batched): `batcher.enqueue(session.threadId, result.text, result.attachment)`

9. Create hook handlers: `const handlers = createHookHandlers(sessionStore, callbacks)`

10. Create hook server: `const server = await createHookServer(config, handlers)`

11. **Reconnect to existing sessions on restart:**
    - Per user decision: "reconnect to existing topics from saved session map + post 'Bot restarted' notification"
    - Get active sessions: `const activeSessions = sessionStore.getActiveSessions()`
    - For each active session with a valid threadId > 0:
      - Send restart notice: `await topicManager.sendBotRestartNotice(session.threadId)`
    - Log how many sessions were reconnected

12. **Start both servers:**
    - Start bot polling: `bot.start()` (this returns immediately in grammY -- it runs in background)
    - Start Fastify: `await server.listen({ port: config.hookServerPort, host: config.hookServerHost })`
    - Log startup complete with port number

13. **Graceful shutdown:**
    - Register `process.on('SIGINT', ...)` and `process.on('SIGTERM', ...)`:
      - `batcher.shutdown()` -- flush pending messages
      - `bot.stop()` -- stop bot polling
      - `await server.close()` -- close Fastify
      - `process.exit(0)`

14. At module level, call `main().catch(err => { console.error('Fatal:', err); process.exit(1); })`
