Create `src/monitoring/transcript-watcher.ts` implementing a class that watches a JSONL transcript file for new entries and emits assistant text messages and token usage updates.

Key implementation details:

1. **Constructor** takes:
   - `filePath: string` (transcript JSONL path from SessionStart payload)
   - `onAssistantMessage: (text: string) => void` callback for posting text to Telegram
   - `onUsageUpdate: (usage: TokenUsage) => void` callback for updating context status

2. **start()** method:
   - Use `statSync` to get initial file size and set `this.filePosition` to skip existing content (we only want new entries after the watcher starts)
   - If file doesn't exist yet, retry with 500ms delay up to 10 times (per Pitfall 3 from research: transcript may not exist until first API call)
   - Use `fs.watch(this.filePath, ...)` for change notifications
   - On each 'change' event, debounce with 100ms timer (per Pitfall 1: fs.watch fires multiple events per write)
   - After debounce, call `readNewLines()`

3. **readNewLines()** private method:
   - Use `openSync`/`readSync`/`closeSync` to read from `this.filePosition` to current file end
   - If current file size < filePosition, the file was truncated/rewritten (compaction) -- reset filePosition to 0 and re-read from start
   - Split read bytes on `\n`, keeping incomplete last line in a `partialBuffer` (per Pitfall 4: lines may split across read boundaries)
   - For each complete line: `JSON.parse` wrapped in try/catch (defensive parsing per MEDIUM confidence JSONL format)
   - Call `processEntry()` for each successfully parsed entry

4. **processEntry(entry: TranscriptEntry)** private method:
   - Skip if `entry.isSidechain === true` (subagent messages)
   - Skip if `entry.type !== 'assistant'` or `entry.message.role !== 'assistant'`
   - Extract text from content: if string, use directly. If array, filter for `type === 'text'` blocks, join with '\n'. Skip `tool_use` and `tool_result` blocks.
   - If extracted text is non-empty, call `this.onAssistantMessage(text)`
   - If `entry.message.usage` exists, call `this.onUsageUpdate(entry.message.usage)`

5. **stop()** method:
   - Close the fs.watch watcher
   - Clear any pending debounce timer
   - Clear retry timer if still retrying file existence

6. **Context percentage helper** (exported standalone function):
   ```typescript
   export const DEFAULT_CONTEXT_WINDOW = 200_000;

   export function calculateContextPercentage(
     usage: TokenUsage,
     contextWindow: number = DEFAULT_CONTEXT_WINDOW
   ): number {
     const totalInputTokens =
       usage.input_tokens +
       usage.cache_creation_input_tokens +
       usage.cache_read_input_tokens;
     return Math.round((totalInputTokens / contextWindow) * 100);
   }
   ```

Use only Node.js built-ins: `import { watch, statSync, openSync, readSync, closeSync, existsSync } from 'node:fs'` and `import type { FSWatcher } from 'node:fs'`. No external dependencies.

Handle all edge cases from research pitfalls: fs.watch multi-fire debounce, partial line buffering, file truncation detection, file-not-yet-exists retry.

**Part A: StatusMessage class** in `src/monitoring/status-message.ts`

A class managing a pinned status message per session topic, updated in place via editMessageText with debouncing.

1. **Constructor** takes:
   - `bot: Bot` (grammy Bot instance)
   - `chatId: number` (Telegram group chat ID)
   - `threadId: number` (topic message_thread_id)

2. **initialize()** method:
   - Send initial status message via `bot.api.sendMessage(chatId, formatStatus(initialData), { message_thread_id: threadId })`
   - Store `this.messageId` from the response
   - Pin the message silently: `bot.api.pinChatMessage(chatId, this.messageId, { disable_notification: true })`
   - Wrap pinChatMessage in try/catch -- non-fatal if bot lacks pin rights (per Pitfall 5)
   - Return the messageId

3. **requestUpdate(data: StatusData)** method:
   - Store `this.pendingData = data`
   - Check elapsed time since last update. If >= 3000ms (minUpdateIntervalMs), call `flush()` immediately
   - Otherwise, set a timer to flush when the interval elapses (per Pitfall 6: separate rate limit for editMessageText)
   - Only one pending timer at a time

4. **flush()** private async method:
   - If no pendingData or no messageId, return
   - Format the status text, compare with `this.lastSentText` -- skip API call if identical (per Pitfall 2: "message is not modified" error)
   - Call `bot.api.editMessageText(chatId, this.messageId, formattedText)`
   - Catch and ignore errors containing "message is not modified"
   - Update `this.lastSentText` and `this.lastUpdateTime`
   - Clear pending timer

5. **updateTopicDescription(description: string)** method:
   - Call `bot.api.editForumTopic(chatId, threadId, { name: description })`
   - Name is limited to 128 chars -- truncate if needed
   - Wrap in try/catch (non-fatal)

6. **destroy()** method:
   - Clear any pending update timer
   - Send a final "Session ended" status update (best-effort)

7. **formatStatus(data: StatusData)** private static or module-level function:
   - HTML formatted status block:
     ```
     [chart emoji] Status: Active
     [bar emoji] Context: 45% used
     [tool emoji] Current: Edit src/foo.ts
     [clock emoji] Duration: 12m
     [counter emoji] Tools: 23 calls | 5 files changed
     ```
   - Use escapeHtml for dynamic values

Import `Bot` from 'grammy', `StatusData` from types/monitoring.js, `escapeHtml` from utils/text.js.

A class that fires periodic summary messages every N milliseconds.

1. **Constructor** takes:
   - `intervalMs: number` (from config.summaryIntervalMs, default 300000 = 5 min)
   - `getSummaryData: () => SummaryData | null` callback that returns current session stats (or null if session inactive)
   - `onSummary: (text: string) => void` callback to post the formatted summary to Telegram

2. **SummaryData interface** (define locally or export):
   ```typescript
   interface SummaryData {
     cwd: string;
     toolCallCount: number;
     filesChanged: number;
     contextPercent: number;
     suppressedCounts: Record<string, number>;
     lastToolName: string;
   }
   ```

3. **start()** method:
   - Set `this.timer = setInterval(...)` with intervalMs
   - On each tick: call getSummaryData(). If null, skip. Otherwise format and call onSummary.
   - Track `this.lastSummaryToolCount` to detect if any activity occurred since last summary (skip summary if no new tool calls)

4. **formatSummary(data: SummaryData)** private method:

   - If suppressedCounts has entries, append: "({count} Read, {count} Glob suppressed)"
   - Skip if no activity since last summary

5. **stop()** method:
   - Clear the interval timer
