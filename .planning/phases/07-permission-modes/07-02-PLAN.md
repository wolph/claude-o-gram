**Mode selection handlers** (ma:, ms:, mf:, mu:):

Each mode handler follows the same pattern:
1. Parse toolUseId from callback_data
2. Look up the session from the pending approval (via approvalManager)
3. Set the mode on PermissionModeManager
4. ALSO approve the current pending tool call (the user chose a mode from this prompt, so also approve it -- per Pitfall 2 in research)
5. Dismiss callback query
6. Edit message to show mode-activated result

For "Same Tool" (`ms:`): Look up the tool name from the pending approval entry. The approval is still pending when the mode button is tapped (the user hasn't approved/denied yet), so retrieve it. Use `approvalManager.decide(toolUseId, 'allow')` which returns the PendingApproval with `.toolName`.

Create a shared helper for all four mode handlers:
```typescript
async function handleModeSelection(
  ctx: Context,
  toolUseId: string,
  mode: PermissionMode,
): Promise<void> {
  // Approve the current tool call AND activate mode
  const pending = approvalManager.decide(toolUseId, 'allow');
  if (!pending) {
    await ctx.answerCallbackQuery({ text: 'This approval has already expired.', show_alert: true });
    return;
  }

  // For same-tool, pass the tool name from the pending approval
  const toolName = mode === 'same-tool' ? pending.toolName : undefined;
  permissionModeManager.setMode(pending.sessionId, mode, toolName);

  await ctx.answerCallbackQuery();

  // Edit message to show mode activation result
  const who = ctx.from?.username ? `@${ctx.from.username}` : ctx.from?.first_name || 'User';
  const modeLabel = { 'accept-all': 'Accept All', 'same-tool': `Same Tool (${pending.toolName})`, 'safe-only': 'Safe Only', 'until-done': 'Until Done' }[mode];
  const originalText = ctx.callbackQuery.message?.text || '';
  const updatedText = `${originalText}\n\n\u26A1 <b>${modeLabel}</b> activated by ${who}`;
  try {
    await ctx.editMessageText(updatedText, {
      parse_mode: 'HTML',
      reply_markup: undefined,
    });
  } catch (err) {
    if (!(err instanceof Error && err.message.includes('message is not modified'))) {
      console.warn('Failed to edit mode activation message:', err instanceof Error ? err.message : err);
    }
  }
}
```

Register four handlers:
