  <files>src/monitoring/topic-status.ts</files>
  <action>
Create a new file `src/monitoring/topic-status.ts` that provides centralized topic status management.

**Exports:**

1. `type TopicStatus = 'green' | 'yellow' | 'red' | 'gray'`

2. `const STATUS_EMOJIS: Record<TopicStatus | 'done', string>` mapping:
   - `green`: `'\u{1F7E2}'` (🟢)
   - `yellow`: `'\u{1F7E1}'` (🟡)
   - `red`: `'\u{1F534}'` (🔴)
   - `gray`: `'\u26AA'` (⚪)
   - `done`: `'\u2705'` (✅)

3. `class TopicStatusManager`:

**Constructor:** Takes `bot: Bot` (grammY Bot instance) and `chatId: number`.

**Private state:**
- `statusCache: Map<number, TopicStatus>` -- tracks last confirmed status per threadId
- `pendingStatus: Map<number, TopicStatus>` -- tracks pending (debounced) status per threadId
- `timers: Map<number, ReturnType<typeof setTimeout>>` -- per-topic debounce timers
- `topicNames: Map<number, string>` -- clean topic names (without emoji) per threadId
- `stickyRed: Set<number>` -- threadIds where red status is sticky (context exhaustion)
- Static readonly `DEBOUNCE_MS = 3000`

**Public methods:**

`registerTopic(threadId: number, cleanName: string): void`
- Store the clean name (no emoji prefix) in `topicNames`
- Call this when a session starts or reconnects

`unregisterTopic(threadId: number): void`
- Delete from `topicNames`, `statusCache`, `pendingStatus`, `stickyRed`
- Clear and delete any timer for this threadId

`setStatus(threadId: number, status: TopicStatus): void`
- If the topic is not registered (not in topicNames), return silently
- If `stickyRed` has this threadId AND status is 'green' or 'yellow', return (red is sticky)
- If `statusCache.get(threadId) === status`, return (no-op)
- Set `pendingStatus.set(threadId, status)`
- If a timer exists for this threadId, clear it
- Set a new timer: `setTimeout(() => void this.flush(threadId), DEBOUNCE_MS)`
- Store timer in `timers`

`setStatusImmediate(threadId: number, status: TopicStatus, cleanName?: string): Promise<void>`
- For immediate (non-debounced) updates: gray sweep and session start
- If cleanName is provided, store it in topicNames
- Cancel any pending timer for this threadId
- Delete from pendingStatus
- If `statusCache.get(threadId) === status`, return (no-op)
- Execute the editForumTopic call directly (same logic as flush)
- On success, update statusCache

`clearStickyRed(threadId: number): void`
- Remove threadId from stickyRed set
- Called when a new turn starts (user sends input), allowing green/yellow transitions again

`getStatus(threadId: number): TopicStatus | undefined`
- Return the current status from statusCache (or pendingStatus if not yet flushed)

`destroy(): void`
- Clear all timers, maps, and sets

**Private methods:**

`async flush(threadId: number): Promise<void>`
- Get status from `pendingStatus`; if none, return
- Delete from `pendingStatus` and `timers`
- If `statusCache.get(threadId) === status`, return (double-check after debounce)
- Get cleanName from `topicNames`; if none, return
- Build full name: `${STATUS_EMOJIS[status]} ${cleanName}`
- Truncate to 128 chars (Telegram limit)
- Try `bot.api.editForumTopic(chatId, threadId, { name: fullName })`
- On success: `statusCache.set(threadId, status)`
- On error: log warning (non-fatal)
- If status is 'red': add threadId to `stickyRed`
